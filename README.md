# NorbitPizza
Pizza ordering software
1. Описание проекта

Проект представляет собой WPF-приложение для пиццерии, предназначенное для визуализации меню и оформления заказа.
Цель — создать удобный интерфейс для клиентов, повысить конверсию заказов и разгрузить сотрудников.

⸻

2. Проблема заказчика
- Нет клиентской части для просмотра меню и оформления заказа.
- Заказы обрабатываются вручную → задержки, ошибки, нагрузка на персонал.
- Сложно анализировать популярность блюд и управлять ассортиментом.

⸻

3. Цели проекта
 1. Упростить процесс выбора и заказа пиццы.
 2. Увеличить количество онлайн-заказов.
 3. Сократить нагрузку на сотрудников.
 4. Обеспечить быстрый доступ к меню.
 5. Подготовить фундамент для дальнейшего развития цифровых сервисов.

⸻

4. Объём работ (Scope)

MVP — минимальный функционал
 - Главная страница со списком пицц (фото + описание + цена).
 - Детализация выбранной пиццы (детейл-вью).
 - Фильтрация меню.
 - Корзина (добавление/удаление товаров, подсчёт суммы).
 - Страница оформления заказа.

Дополнительно / будущие версии
 - Процесс оплаты.
 - CRM для контроля заказов и базы данных.
 - Регистрация пользователей.
 - Программа лояльности.
 - Автоматическое определение самых популярных позиций (“хиты продаж”).
 - Тёмная тема и локализация интерфейса.

⸻

5. Приоритеты разработки (MVP First)
 1. Отображение меню и структура данных.
 2. Корзина + пересчёт итоговой суммы.
 3. Оформление заказа.
 4. Детализация пиццы.
 5. Фильтры.

Такой порядок гарантирует выполнение обязательной части к дедлайну.

⸻

6. Архитектура и масштабируемость
 - Использована MVVM-модель, что обеспечивает разделение логики, данных и интерфейса.
 - Код написан с учётом ООП-концептов, компоненты изолированы.
 - Структура проекта включает чётко выделенные фолдеры: Models, ViewModels, Views, Services.
 - API и база данных спроектированы для расширяемости без переписывания существующей логики.
 - База данных (БДшка) хранит:
 - продукты и категории,
 - пользователей,
 - заказы и позиции заказа,
 - историю транзакций.
 - Возможна интеграция новых таблиц, каскадное удаление и аналитика без слома системы.

⸻

7. Оценка трудозатрат

Этап Часы Исполнитель
Выбор платформы, дизайн/шаблоны 3 Dev
Разработка БД 3 Dev
Разработка API 10 Dev
Главная страница 8 Dev
Корзина/оформление заказа 6 Dev
Мануальное тестирование 2–3 QA
Документация и аналитика 4–6 BA

Итого: 36–39 часов разработки + 2–3 часа тестирования + 4–6 часов аналитики

8. Стоимость разработки (3 дня)

Команда
 - 3 разработчика
 - 1 бизнес-аналитик
 - 1 тестировщик

Средние ставки (USD/день)

Роль Ставка Дни
Dev 200$ 3
BA 180$ 3
QA 150$ 3

Итоговые затраты
 - Разработчики: 3 × 200 × 3 = 1800$
 - BA: 180 × 3 = 540$
 - QA: 150 × 3 = 450$

Итого: ~2800–3200$

Минимальный MVP
 - 1 разработчик × 3 дня → 600$
 - QA (1 день) → 150$
 - BA (1–1.5 дня) → 200–300$

Итого MVP: ~1000$

11. Тестирование
 - Мануальное тестирование — основное, занимает 2–3 часа.
 - Юнит-тесты не входят в MVP, потребуют отдельной разработки API, БД и UI.
 - Проверяется: работа корзины, оформление заказа, деталь продукта, фильтры.
